Most useful formulas for administrators
FunctionDescriptionSyntax and Formula
MONTH-YEARIndicates the month and year a transaction was posted in text format, such as MAY-2025.Syntax: TEXT(value, format_text)=TEXT([@[Journal Date]],"MMMM-YYYY")
FISCAL YEARIndicates the University fiscal year in which a transaction was posted. Columbia’s fiscal year runs July 1 through June 30.Syntax: IF(logical_test, value_if_true, value_if_false)=IF(MONTH([@[Journal Date]])>=7,YEAR([@[Journal Date]])+1,YEAR([@[Journal Date]]))
FISCAL QUARTERIndicates the fiscal quarter based on the University fiscal calendar. Example: Q1 = July–September.Syntax: IF(MONTH(date)>=7,...)=IF(MONTH([@[Journal Date]])>=7,"Q"&INT((MONTH([@[Journal Date]])-7)/3)+1,"Q"&INT((MONTH([@[Journal Date]])+5)/3)+1)
ACADEMIC TERMIndicates the academic term in which the transaction occurred: Spring Jan–May, Summer Jun–Aug, Fall Sep–Dec.Syntax: IF(logical_test, value_if_true, value_if_false)=IF(MONTH([@[Journal Date]])>=9,"Fall",IF(MONTH([@[Journal Date]])>=6,"Summer","Spring"))
DAYS FROM INVOICE TO POSTINGShows how many days elapsed between the invoice date and the journal posting date.Syntax: IFERROR(value, value_if_error)=IFERROR([@[Journal Date]]-[@[Invoice Date]],"")
SUMIFSAdds values that meet multiple conditions.Syntax: SUMIFS(sum_range, criteria_range1, criteria1, ...)=SUMIFS(E:E,A:A,"Travel",B:B,"FY2026")
COUNTIFSCounts cells meeting multiple conditions.Syntax: COUNTIFS(criteria_range1, criteria1, ...)=COUNTIFS(A:A,"AP",B:B,"FY2026")
UNIQUEReturns unique values from a range.Syntax: UNIQUE(array, [by_col], [exactly_once])=UNIQUE(A2:A500)
SORT + UNIQUEReturns a clean alphabetical list of distinct values. Useful for vendor lists, project lists, account lists, department lists, and pivot cleanup.Syntax: SORT(UNIQUE(array))=SORT(UNIQUE(A2:A500))
XLOOKUPLooks up a value and returns a related result from another range.Syntax: XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found])=XLOOKUP(A2,Projects[Project],Projects[Description],"Not found")
AGGREGATEReturns aggregate calculations with options to ignore errors or hidden rows.Syntax: AGGREGATE(function_num, options, array)=AGGREGATE(9,6,E:E)
TODAYReturns the current date.Syntax: TODAY()=TODAY()
LEFTReturns characters from the beginning of text. Best when a department, fund, project prefix, or category code starts the string.Syntax: LEFT(text, [num_chars])=LEFT("happy",2)
Concrete example: From happy, return the first 2 characters → ha.
Next level: Use =LEFT([@ChartString],5) to extract a leading department or code segment from a longer ChartString.
RIGHTReturns characters from the end of text. Best when the useful identifier is a suffix, year, final digits, or trailing code.Syntax: RIGHT(text, [num_chars])=RIGHT("happy",2)
Concrete example: From happy, return the last 2 characters → py.
Next level: Use =RIGHT([@Project],4) to isolate the trailing project sequence or reporting suffix.
COUNTACounts cells that are not empty.Syntax: COUNTA(value1, [value2], ...)=COUNTA(A2:A500)
IFReturns one value if a condition is true and another if false.Syntax: IF(logical_test, value_if_true, value_if_false)=IF(A2>B2,"Over","Within")
ANDReturns TRUE if all conditions are true.Syntax: AND(logical1, [logical2], ...)=AND(A2>0,B2<10)
ORReturns TRUE if at least one condition is true.Syntax: OR(logical1, [logical2], ...)=OR(A2="AP",A2="Payroll")
IF + ORReturns a result when at least one listed condition is true. Useful for flagging rows that meet any review trigger.Syntax: IF(OR(logical1, logical2), value_if_true, value_if_false)=IF(OR(A2="AP",A2="Payroll"),"Review","OK")
COUNTA + UNIQUECounts how many distinct values appear in a range.Syntax: COUNTA(UNIQUE(array))=COUNTA(UNIQUE(A2:A500))
INDEX + MATCHLooks up a value by matching a row position, then returning the related result from another range.Syntax: INDEX(return_range, MATCH(lookup_value, lookup_range, 0))=INDEX(C:C,MATCH(A2,B:B,0))

Popular Functions

Core formulas every administrator should know
FunctionDescriptionSyntax and Formula
IFEvaluates a condition and returns one value if true and another if false.Syntax: IF(logical_test, value_if_true, value_if_false)=IF(A2>1000,"Review","OK")
ANDReturns TRUE only when all specified conditions are true.Syntax: AND(logical1, [logical2], ...)=AND(A2>0,B2="Approved")
ORReturns TRUE when at least one condition is true.Syntax: OR(logical1, [logical2], ...)=OR(A2="AP",A2="PCard")
SUMAdds values across a range of cells.Syntax: SUM(number1, [number2], ...)=SUM(E2:E500)
AVERAGECalculates the average of a set of numbers.Syntax: AVERAGE(number1, [number2], ...)=AVERAGE(E2:E500)
COUNTCounts cells that contain numbers.Syntax: COUNT(value1, [value2], ...)=COUNT(A2:A500)
COUNTACounts cells that are not empty.Syntax: COUNTA(value1, [value2], ...)=COUNTA(A2:A500)
CONCATCombines text from multiple cells.Syntax: CONCAT(text1, [text2], ...)=CONCAT(A2," - ",B2)
LEFTExtracts characters from the left side of a text string. Use this when the meaningful part of a code starts at the beginning.Syntax: LEFT(text, [num_chars])=LEFT("happy",2)
Concrete example: From happy, return the first 2 characters → ha.
Next level: Use =LEFT([@Project],2) to pull the first two characters of a project code for grouping or review.
RIGHTExtracts characters from the right side of a text string. Use this when the meaningful part of a code appears at the end.Syntax: RIGHT(text, [num_chars])=RIGHT("happy",2)
Concrete example: From happy, return the last 2 characters → py.
Next level: Use =RIGHT([@Invoice ID],4) to pull the final four characters of an invoice ID for quick matching or audit review.
TODAYReturns the current date.Syntax: TODAY()=TODAY()
VLOOKUPSearches the first column of a table and returns a related value.Syntax: VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])=VLOOKUP(A2,Reference!A:D,4,FALSE)
XLOOKUPModern lookup function that can search left, right, exact, approximate, or custom fallback results.Syntax: XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found])=XLOOKUP(A2,VendorList[ID],VendorList[Name],"Not found")
SUMIFSums values based on one condition.Syntax: SUMIF(range, criteria, [sum_range])=SUMIF(A:A,"Travel",E:E)
SUMIFSSums values based on multiple conditions.Syntax: SUMIFS(sum_range, criteria_range1, criteria1, ...)=SUMIFS(E:E,A:A,"Travel",B:B,"FY2026")

FunctionDescriptionSyntax and Formula

Logical Functions

Make decisions inside formulas
FunctionDescriptionSyntax and Formula
ANDReturns TRUE if all conditions are true.Syntax: AND(logical1, [logical2], ...)=AND(A2>0,B2<10)
FALSEReturns the logical value FALSE.Syntax: FALSE()=FALSE()
IFReturns one value if a condition is true and another if false.Syntax: IF(logical_test, value_if_true, value_if_false)=IF(A2>B2,"Over","Within")
IFERRORReturns a fallback value when a formula produces an error.Syntax: IFERROR(value, value_if_error)=IFERROR(A2/B2,0)
IFNAReturns a fallback value when a formula produces #N/A.Syntax: IFNA(value, value_if_na)=IFNA(XLOOKUP(A2,D:D,E:E),"Missing")

NOTReverses TRUE and FALSE.Syntax: NOT(logical)=NOT(A2="Closed")
ORReturns TRUE if at least one condition is true.Syntax: OR(logical1, [logical2], ...)=OR(A2="AP",A2="Payroll")
SWITCHEvaluates a value against a list and returns a matching result.Syntax: SWITCH(expression, value1, result1, ..., [default])=SWITCH(A2,"AP","Voucher","PCARD","Card","Other")
TRUEReturns the logical value TRUE.Syntax: TRUE()=TRUE()

Combination Formulas

High-impact formulas built by combining functions
FunctionDescriptionSyntax and Formula
IF + ORFlags a row when at least one listed condition is true.Syntax: IF(OR(condition1, condition2), value_if_true, value_if_false)=IF(OR(A2="AP",A2="Payroll"),"Review","OK")
IF + ANDFlags a row only when all listed conditions are true.Syntax: IF(AND(condition1, condition2), value_if_true, value_if_false)=IF(AND(E2>5000,B2="FY2026"),"High FY26 Charge","OK")
IFERROR + XLOOKUPRuns a lookup and returns a clean fallback message if the lookup fails.Syntax: IFERROR(XLOOKUP(...), fallback)=IFERROR(XLOOKUP(A2,Projects[Project],Projects[Description]),"Missing Project")
TRIM + UPPERCleans spacing and standardizes text to uppercase for cleaner matching.Syntax: UPPER(TRIM(text))=UPPER(TRIM([@[Transaction Description / Vendor Name]]))
COUNTA + UNIQUECounts distinct values in a list, such as unique vendors, accounts, or projects.Syntax: COUNTA(UNIQUE(array))=COUNTA(UNIQUE(A2:A500))
SORT + UNIQUECreates a sorted distinct list from messy report data.Syntax: SORT(UNIQUE(array))=SORT(UNIQUE([Vendor ID]))
INDEX + MATCHMatches a lookup value and returns the corresponding result from another range.Syntax: INDEX(return_range, MATCH(lookup_value, lookup_range, 0))=INDEX(C:C,MATCH(A2,B:B,0))
TEXT + EOMONTHConverts a date into a clean month-end reporting label.Syntax: TEXT(EOMONTH(date,0), format_text)=TEXT(EOMONTH(A2,0),"MMMM YYYY")
ABS + IFFlags transactions based on magnitude regardless of whether the value is positive or negative.Syntax: IF(ABS(number)>threshold, value_if_true, value_if_false)=IF(ABS(E2)>10000,"Material","Standard")
SUMIFS + TODAYSummarizes values through the current date for rolling review.Syntax: SUMIFS(sum_range, date_range, "<="&TODAY())=SUMIFS(E:E,S:S,"<="&TODAY())
LEFT + TEXTBEFOREPulls a clean segment from the beginning of a structured label before further shortening it.Syntax: LEFT(TEXTBEFORE(text, delimiter), num_chars)=LEFT(TEXTBEFORE(A2,"-"),5)
